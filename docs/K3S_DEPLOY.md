# Deploying AstracatDNS on K3s / Kubernetes

This guide explains how to deploy AstracatDNS into a K3s or Kubernetes cluster using the adapted Dockerfile and manifests.

## Prerequisites

*   **Kubernetes Cluster**: A running K3s or K8s cluster.
*   **kubectl**: Configured to talk to your cluster.
*   **Docker**: To build the container image.

## Quick Start

We have provided a helper script to build and deploy in one go:

```bash
./deploy_k3s.sh
```

This script will:
1.  Build the Docker image `astracat-dns:latest`.
2.  Import it into K3s (if running locally) or you must ensure your cluster can pull it.
3.  Apply the Kubernetes manifests.

## Manual Deployment Steps

### 1. Build the Docker Image

Since AstracatDNS is a custom application, you need to build the image first.

```bash
docker build -t astracat-dns:latest .
```

> **Note for K3s Users:** If you are running K3s locally, you need to import the image so K3s can see it:
> ```bash
> docker save astracat-dns:latest | sudo k3s ctr images import -
> ```
> Or, if using a remote cluster, push the image to a container registry (Docker Hub, GHCR, etc.) and update `k8s/deployment.yaml` with the new image tag.

### 2. Configuration

The configuration is stored in `k8s/configmap.yaml`. This file is automatically generated from your local `config.yaml` and `hosts` file if you use the generation command, but for now, it includes the default configuration.

If you need to change settings:
1.  Edit `k8s/configmap.yaml`.
2.  Apply the changes: `kubectl apply -f k8s/configmap.yaml`
3.  Restart the pods: `kubectl rollout restart deployment astracat-dns`

### 3. Apply Manifests

Run the following commands to deploy:

```bash
kubectl apply -f k8s/configmap.yaml
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
```

### 4. Verify Deployment

Check if the pods are running:

```bash
kubectl get pods -l app=astracat-dns
```

Check the service IP to access DNS:

```bash
kubectl get svc astracat-dns
```

By default, the service is of type `LoadBalancer`. In K3s, this will likely assign the Node's IP address.

## Accessing the Service

*   **DNS (UDP/TCP):** Port 53
*   **DoH:** Port 443
*   **DoT:** Port 853
*   **Admin Panel:** Port 8080
*   **Metrics:** Port 9090

To test DNS:
```bash
dig @<SERVICE-IP> google.com
```

## Persistence

The cache is currently configured to use an `emptyDir` volume, which means the cache is cleared when the Pod restarts. If you require persistent caching across restarts, you should modify `k8s/deployment.yaml` to use a `PersistentVolumeClaim`.
