# BalanceDNS - Complete System Overview

## 🎯 What Was Done

### 1. Fixed Critical Server Crashes (5 issues)

| Issue | Problem | Solution |
|-------|---------|----------|
| **Panic abort** | Server died instantly on any panic | Changed to `panic = "unwind"` in Cargo.toml |
| **No panic guards** | Thread panics killed server | Added `catch_unwind()` to all thread handlers |
| **No session timeouts** | Connections stayed open forever | Added 30s idle timeout + 1000 query limit |
| **Dangerous unwrap()** | Could panic on malformed packets | Proper error handling in cache.rs |
| **Poor logging** | Hard to debug issues | Changed critical logs from debug→warn |

**Result:** Server now survives panics, auto-recovers, and won't crash from bad packets.

### 2. Created Automatic Monitoring System (4 layers)

```
Layer 1: Systemd Watchdog (60s timeout)
         ↓
Layer 2: Health Check Timer (every 30s)
         ↓
Layer 3: Auto-Restart (with cooldown)
         ↓
Layer 4: Monitoring Scripts (real-time)
```

**What gets detected:**
- ✅ Process crash → Restart in 2s
- ✅ Process hang → Kill after 60s
- ✅ DNS unresponsive → Restart after 30s
- ✅ Memory leak → 2GB limit enforced
- ✅ Too many connections → 4096 limit

### 3. Created Automatic Installer

**One command does everything:**
```bash
sudo ./install.sh
```

**Installs:**
1. ✅ Compiles BalanceDNS from source
2. ✅ Creates system user & directories
3. ✅ Generates configuration
4. ✅ Installs systemd service + watchdog
5. ✅ Installs health check monitoring
6. ✅ Starts & verifies everything

## 🛡️ Failure Scenarios

### Scenario 1: Server Crashes
```
1. Process crashes
2. systemd detects immediately
3. Restarts after 2 seconds
4. Health check verifies it's working
5. ✅ Total downtime: ~3 seconds
```

### Scenario 2: Server Hangs (Deadlock)
```
1. Process stops responding
2. Watchdog waits 60 seconds
3. Sends SIGABRT to kill process
4. systemd restarts it
5. Health check verifies recovery
6. ✅ Total downtime: ~65 seconds
```

### Scenario 3: DNS Becomes Unresponsive
```
1. Health check runs every 30s
2. Tries DNS query (2 attempts)
3. Detects unresponsive server
4. Restarts service
5. Verifies DNS is working
6. ✅ Total downtime: ~35 seconds
```

### Scenario 4: Thread Panics
```
1. Query causes panic in thread
2. catch_unwind() catches it
3. Thread exits gracefully
4. Server keeps running
5. Error logged for debugging
6. ✅ No downtime at all!
```

## 🔧 Configuration

### Quick Edit
```bash
nano /etc/balancedns.toml
systemctl restart balancedns
```

### Key Settings

```toml
[server]
udp_listen = "YOUR_IP:53"     # Main DNS port
tcp_listen = "YOUR_IP:53"     
dot_listen = "YOUR_IP:853"    # DNS-over-TLS
doh_listen = "YOUR_IP:443"    # DNS-over-HTTPS

[cache]
max_size = 100000              # Cache entries
ttl_seconds = 7200             # Default TTL

[global]
max_tcp_clients = 4096         # Max TCP connections
threads_udp = 8                # UDP worker threads
```

### Adjust Monitoring

```bash
# Health check frequency
nano /etc/systemd/system/balancedns-healthcheck.timer
# Change: OnUnitActiveSec=30

# Watchdog timeout
nano /etc/systemd/system/balancedns.service  
# Change: WatchdogSec=60

# Then reload
systemctl daemon-reload
systemctl restart balancedns
```

### Common Issues

**"Service failed to start"**
- Check config: `/usr/sbin/balancedns --config /etc/balancedns.toml --test`
- Check ports: `ss -tulpn | grep :53`
- Check logs: `journalctl -u balancedns -n 50`

**"DNS not responding"**
- Verify IP: Make sure config has correct server IP
- Check firewall: Allow port 53 (UDP+TCP)
- Test locally: `dig @127.0.0.1 localhost A`

**"Too many restarts"**
- Check: `systemctl show balancedns --property=NRestarts`
- Increase cooldown in healthcheck script
- Check upstream DNS servers are reachable
