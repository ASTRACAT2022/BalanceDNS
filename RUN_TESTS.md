# Тестирование DNS-сервера

## Запуск DNS-сервера

Сначала убедитесь, что Go установлен и запустите DNS-сервер:

```bash
cd /home/astracat/Astracat-DNS-Resolver
go run main.go
```

Сервер будет слушать на порту 5053.

## Тестирование с помощью dig

### Простое тестирование
```bash
# Сделать скрипт исполняемым
chmod +x test_dns.sh

# Запустить тестирование
./test_dns.sh
```

### Проверка ответов DNS
```bash
# Простой запрос
dig @127.0.0.1 -p 5053 google.com A +short

# Подробный запрос
dig @127.0.0.1 -p 5053 google.com A +multiline

# Проверка различных типов записей
dig @127.0.0.1 -p 5053 github.com A
dig @127.0.0.1 -p 5053 github.com AAAA
dig @127.0.0.1 -p 5053 github.com MX
```

## Нагрузочное тестирование

### Скриптовое тестирование
```bash
# Сделать скрипт исполняемым
chmod +x load_test.sh

# Запустить нагрузочное тестирование (10 потоков, 10 запросов на поток)
./load_test.sh 10 10

# Запустить с более высокой нагрузкой (50 потоков, 20 запросов на поток)
./load_test.sh 50 20
```

### Тестирование производительности на Go
```bash
# Запустить тестирование производительности
go run performance_test.go
```

## Мониторинг производительности

DNS-сервер также запускает сервер метрик на порту 9090, который можно использовать для мониторинга:

```bash
# Просмотр метрик
curl http://127.0.0.1:9090/metrics
```

## Ожидаемые результаты

После моих исправлений DNS-сервер должен:

1. Возвращать корректные DNS ответы вместо NOERROR без записей
2. Кэшировать ответы для улучшения производительности
3. Обрабатывать высокую нагрузку (до нескольких тысяч QPS в зависимости от конфигурации)
4. Поддерживать параллельные запросы без блокировок
